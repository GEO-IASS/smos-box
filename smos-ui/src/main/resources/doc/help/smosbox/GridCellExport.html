SMOS Grid Cell Export

SMOS Grid Cell Export

The grid cell export tool writes the data of a selection of grid cells into a CSV (character separated values) file. The values are separated by a semicolon.

Export tool from within VISAT

Input: The selected product within VISAT can be for the export. Alternatively the export tool can do an directory scan to export data from multiple SMOS products. This directory scan can be either flat, where only products that reside in the selected directory are found, or recursive, where also all sub directories are searched for products.

Region of interest: There are three different possibilities to specify for which grid cells data should be exported.

    - A ROI containing a figure which can defined on any band.
     - The grid cells from all selected pin of the currently selected product.
     - A specified longitude/latitude box.

Output: The CSV file to which the data is written. Please be aware that the amount of data can be huge, if the region of interest contains many grid cells.

Command line export tool

The command line tool for grid cell exporting can executing using the "export-grid-cells.sh" or "export-grid-cells.bat" which resides in the BEAM_HOME/bin directory.

The grid cells for which data should be exported can be specified by defining a point with its longitude and latitude:
export-grid-cells.sh -point 120 30 smosproduct.HDR

Alternatively a longitude/latidue box can be given:
export-grid-cells.sh -box -120 -100 30 40 smosproduct.HDR

Only one these both options can be used.

The file to which the data should be written can specified using the -o parameter: option. Otherwise stdout is used.
export-grid-cells.sh -point 120 30 -o grid_cell_data.csv smosproduct.HDR

Multiple SMOS products from which data should be exported can be specified. Then usual shell pattern can be used to specify multiple files:
export-grid-cells.sh -point 120 30 -o grid_cell_data.csv SMOS_archive/SM_*/SM*.HDR
