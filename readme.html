About the BEAM SMOS-Box Version 0.5

Introduction

The BEAM SMOS-Box project has been brought into life in order to support users of data acquired by ESA's Soil Moisture and Ocean Salinity (SMOS) mission.

Version 0.5 of the SMOS-Box is preliminary software released and distributed for the purpose of obtaining user feedback and additional user requirements at an early stage of development.

The SMOS-Box is developed under ESA contract initiated by Steven Delwart (ESTEC) and coordinated by Peter Regner (ESRIN). The SMOS-Box software is developed by Brockmann Consult (BEAM Team).

Capabilities

Supported Products

Version 0.5 of the SMOS-Box is capable of reading SMOS user data products complying with Schemas Release 2009-02-10_v04-03-00:

Level-1c Browse Full and Dual Polarisation
All measurements contained in the BT_Data_Type record are converted into BEAM bands.
Level-1c Science Full and Dual Polarisation
All measurements contained in the BT_Data_Type record are converted into BEAM bands. For each band, the user can choose whether the band data should be provided by a snapshot or a browse-equivalent which is computed on-the-fly. The latter is the default.
Level-2 Ocean Salinity UDP
All measurements provided in the Grid_Point_Data_Type record and the enclosed record types are converted into BEAM bands.
Level-2 Surface Moisture UDP
All measurements provided in the Grid_Point_Data_Type record and the enclosed record types are converted into BEAM bands.

Tools

Version 0.5 of the SMOS-Box also includes four SMOS-specific tools:

Grid point brightness temperature data table
All measurements acquired for the grid point determined by either the mouse pointer or a selected pin are displayed in a table.
Grid point brightness temperature flag matrix
The flags associated with the measurements made for the grid point determined by either the mouse pointer or a selected pin are displayed in a flag matrix.
Grid point brightness temperature chart
The brightness temperatures acquired for the grid point determined by either the mouse pointer or a selected pin are displayed in a chart.
Snapshot information tool
Information for all snapshots contained in a Level-1c Science product can be browsed with a slider. When synchronised with the current view, the slider can be used to select a snapshot providing the band data for the view.

Limitations

Version 0.5 of the SMOS-Box has the following limitations:

  • All SMOS data products are represented as global products covering the whole surface of the Earth. Therefore SMOS products do not have a geographical boundary.
  • Due to the representation of SMOS products as a global product, it is not possible to draw lines or shapes crossing the anti-meridian.
  • Exporting a displayed SMOS image results in an out-of-memory error. The VISAT application must be restarted in order to recover from this error.
  • Some map projections do not work with SMOS products. Those projections which do work need too much time in order to complete the calculations.
  • Exporting a SMOS product as BEAM-DIMAP, HDF5 or GeoTIFF does not work.
  • Statistics, scatter plots, histograms and profile plots use image pixel data rather than SMOS grid point data.
  • The brightness temperature measurements are not transformed into the earth-fixed frame yet.
  • The graphical user interface occasionally is less responsive while working with SMOS products.
  • The default coloring used for displaying Level-2 bands is not adequate yet.
